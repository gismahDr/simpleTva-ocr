# Root Cause Analysis

Use this skill whenever fixing bugs, exceptions, unexpected behavior, failing tests, compilation errors, or data inconsistencies.

## Goal

Find the actual cause of the problem before making any code changes.

## Rules

1. Reproduce the issue whenever possible.
2. Read all relevant code before editing.
3. Identify the root cause, not just the visible symptom.
4. Explain the root cause before proposing a fix.
5. Search for similar code paths that may be affected.
6. Prefer fixing the cause over adding workarounds.

## Forbidden

* Blind code changes.
* Guessing.
* Adding code without understanding the failure.
* Fixing multiple unrelated issues at the same time.

## Deliverable

Before editing code, provide:

* Root cause
* Affected files
* Proposed fix
* Potential risks
